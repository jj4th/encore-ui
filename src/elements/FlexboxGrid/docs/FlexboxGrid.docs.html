The grid system described below is based on CSS classes.
You can view the documentation for the attribute-based grid system via this link. The visual styling of all examples on this page are for demonstration purposes.
Rows and columns will not have any visual styling applied to them in the consumed CSS.

Overview

The grid system described on this page makes use of flexbox CSS properties and is designed to be mobile first.

Defining a Row

Use the .grid-row CSS class to define a row nesting context. This will automatically apply a row-based flexbox alignment to the assigned element.

  • Element children will automatically be aligned horizontally along the X-axis, from left to right.
  • Element children will wrap if they cannot fit horizontally in the row.

Do

  • Place content in a column of the row.

Don't

  • Do not place content directly within a row.
  • Do not change horizontal margins on a row.
  • Do not change horizontal padding on a row.
  • Do not add borders to a row.

Define Columns

Use the .grid-col class to define a column nesting context. This will automatically apply a column-based flexbox alignment to the assigned element.

  • Children of a column will automatically align vertically along the Y-axis, from top to bottom.
  • Children will share the row width equally, by default.
    Useful for symmetrical layouts.

Do

  • Apply a column span (described below) for specfic column widths.

Don't

  • Do not change horizontal padding on a column.
  • Do not change horizontal margin on a column.
  • Do not add borders to a column.

Define Column Spans

Static Column Spans

Use the .span-N CSS class (where 1 ≤ N ≤ 12) to define columns that are a predefined fraction of the row width.

  • If the total column span exceeds a total of 12, the remaining columns will wrap to the next line, but within the same row.
  • The .span-N class is not responsive and will apply to all available breakpoints.

Don't

  • Do not override width or min-width properties of a column span.

Responsive Column Spans

Breakpoints are based on em units instead of px to retain proper responsiveness for varying window zoom levels. If the window zoom is increased, the page contents enlarge and less space is available for content on the page. As font-size increases, breakpoints trend toward the mobile breakpoints to account for reduced space.

  • The .span-B-N CSS class variants can be used for responsive layouts.
    • B is a breakpoint of xs, sm, md, lg, to xl to match the minimum breakpoint needed to achieve the desired column span.
    • N follows the same rules as static column spans defined above.

Nesting Rows and Columns

You can nest rows can columns as deep as you wish. Gutters will automatically align properly.

Do

  • Beginning with a row element, alternate between rows and columns as you nest.

Offsetting Columns

Use the .offset-N CSS class (where 1 ≤ N ≤ 11) to define where the column span begins.

  • The .offset-N CSS class is not resposive.
  • The .offset-B-N CSS class variants provide responsive functionality. The same rules apply as documented in responsive columns.

Ordering Columns

Use the .order-N CSS class (where 1 ≤ N ≤ 12) to redefine column layout order.

  • The .order-N CSS class is not resposive.
  • The .order-B-N CSS class variants provide responsive functionality. The same rules apply as documented in responsive columns.

Going Gutterless

There will be times where having gutters is undesirable for the layout of elements on the page. For these cases, we have .flex-* equivalents of .grid-* classes to provide 12-column functionality without gutters.

Defining a Gutterless Row

Use the .flex-row CSS class to define a gutterless, row nesting context. This will automatically apply a row-based flexbox alignment to the assigned element.

Do

  • You may add content within a .flex-row.

Don't

  • Do not use .grid-col elements within a gutterless row.

Defining a Gutterless Column

Use the .flex-col class to define a gutterless, column nesting context. This will automatically apply a column-based flexbox alignment to the assigned element.

Do

  • You may apply custom padding to a gutterless column.

Don't

  • Do not use gutterless columns within .grid-row elements.
